    Glossary for Object-Oriented Programming

    Class: are a list of attributes

    Class method: is a method that’s shared among all objects.

    Constructor: is a special method used to create and initialize an object of a class.

    Design Patterns Used: class and behavioral the most essential part of Software Engineering, as they provide the general repeatable solution to a commonly occurring problem in software design

    Façade: is an object that serves as a front-facing interface masking more complex underlying or structural code.

    Factory: is an object for creating other objects formally a factory is a function or method that returns objects of a varying prototype or class from some method call, which is assumed to be "new"

    Fixture: provide a fixed baseline so that tests execute reliably and produce consistent, repeatable, results.

    Instantiation: process of taking a class in creating an object from it, that you can use in your program.

    Method: is a procedure that represents the behavior of a class and is associated with the objects of that class. Runs the code on and off.

    Namespace: is a collection of currently defined symbolic names along with information about the object that each name references.

    Object: are collections of information that are treated as a singular entity.

    Property: is a built-in function that creates and returns a property object.

    Static: method that belongs to the class.

    Type cast: convert a variable value from one type to another.

    Type hint: feature of python that allows you to explicitly declare the data type of a variable when declaring it.

    Unit test: is a type of software testing where individual units or components of a software are tested.
